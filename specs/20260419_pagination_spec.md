# レッスン一覧のページネーション

## 概要
- やりたいこと
  - レッスン一覧の描画件数が多いと動作が重くなるため、ページネーション（追加読み込み方式）を導入して初期描画速度を改善する。
- 背景・理由
  - 現在はフィルタリング後の全件を一括で DOM に描画しており、データ件数（数百〜千件以上）が増えるにつれてスクロールやフィルター操作のレスポンスが悪化している。

## 仕様
### 機能要件
- **初期表示件数**: 50件。
- **追加読み込み**: 一覧の最下部に「もっと見る」ボタンを配置し、タップで次の50件を追加表示する。
- **全件表示済み**: すべてのデータが表示されている場合は「もっと見る」ボタンを非表示にする。
- **フィルター連動**: フィルター条件が変更された際、表示ページを1番目（最初の50件）にリセットする。
- **表示件数の更新**: 「{全件数} 件中 {表示件数} 件を表示」のような形式で現在の表示状態をユーザーに伝える。

### UI/メッセージ
- ボタンラベル: 「もっと見る」
- 表示件数ラベル: `{filteredData.length} 件中 {displayCount} 件を表示`
- 完了メッセージ: (なし)
- エラーメッセージ: (なし)

### 制約・前提
- 既存の関連機能/コード: `src/view.html` (React コンポーネント)
- 使用する既存のDAO/Utils: なし (フロントエンドのみの変更)

## 実装計画
### 使用するクラス・ファイル
- WebView の HTML: `src/view.html`

### 処理フロー
1. `src/view.html` の `App` コンポーネントに `displayLimit` (初期値 50) の `useState` を追加する。
2. `filteredData` を `slice(0, displayLimit)` して描画用のデータを算出する。
3. リストの末尾に、`displayLimit < filteredData.length` の場合に「もっと見る」ボタンを表示する。
4. 「もっと見る」ボタン押下時に `displayLimit` を 50 増やす。
5. `filters` が変更された際に `displayLimit` を 50 にリセットする `useEffect` を追加する。
6. 件数表示部分（現在は `{filteredData.length} 件表示`）を `{filteredData.length} 件中 {displayCount} 件を表示` に更新する。

### 技術的な判断・注意点
- **なぜ「もっと見る」方式か**: モバイル端末での操作性を考慮し、ページ番号による遷移よりもスクロールの流れを止めない追加読み込み方式を採用する。
- **仮想スクロールの見送り**: 実装の単純さと、50件ずつの描画であれば React の再描画コストは十分に許容範囲内であることから、複雑な仮想リストライブラリの導入は避ける。
