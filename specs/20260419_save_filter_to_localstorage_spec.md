# 検索条件の localStorage 保存

## 概要
- ユーザーが選択した検索条件（フィルタ）をブラウザの `localStorage` に保持する。
- 画面をリロードしたり、一度閉じたりした後でも、前回の検索条件が維持されるようにする。

## 仕様
### 機能要件
- フィルタ条件（曜日、時間帯、店舗、ジャンル、レベル、女性限定）が変更されるたびに `localStorage` に保存される。
- 画面の初期読み込み時に `localStorage` から保存された条件を読み込み、初期値として設定する。
- 「クリア」ボタンが押された際は、`localStorage` のデータもクリアされる。
- 保存されるキー名は `noa-lesson-filters` とする。
- データ形式は JSON 文字列とする。

### UI/メッセージ
- メニュー名: なし（バックエンド処理のみ）
- 確認メッセージ: なし
- 完了メッセージ: なし
- エラーメッセージ: なし（データのパース失敗時はデフォルト値を使用し、エラーはコンソールに出力する）

### 制約・前提
- `localStorage` が利用可能なブラウザ環境であること（Google Apps Script の Web App として配布され、モダンブラウザで実行されることを想定）。
- 既存の `filters` ステート（React の `useState`）をベースに実装する。

## 実装計画
### 使用するクラス・ファイル
- View: `src/view.html`

### 処理フロー
1. フィルタの初期状態を `localStorage` から取得するロジックを `getInitialFilters` 関数として追加する。
2. `filters` ステートの初期値を上記関数の戻り値にする。
3. `useEffect` を追加し、`filters` が変更されるたびに `localStorage` へ保存する。
4. `clearAllFilters` 関数内で `localStorage` を削除する。

### 技術的な判断・注意点
- 保存するデータ量（フィルタ条件のみ）は非常に小さいため、`localStorage` で十分である。
- ジャンル名などの静的なリストが変更された場合に不整合が起きないよう、読み込み時にフィルタオプションと照合するか、存在しない値は無視されるように既存のフィルタロジックを活用する（既存のフィルタロジックは `data` に含まれない値は自然と除外されるため安全）。
