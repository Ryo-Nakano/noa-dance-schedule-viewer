# 「今日」フィルター機能

## 概要
- フィルターの選択肢に「今日」という動的な項目を追加する。
- ユーザーが「今日」を選択すると、常にその時点での当日のレッスンが表示されるようになる。
- 背景: `localStorage` に「月曜日」などの固定の曜日を保存してしまうと、翌日開いた際にもそのまま「月曜日」が表示されてしまう問題がある。「今日」という動的なフィルター値を持たせることで、いつアプリを開いても最新の「今日」の情報を優先して表示可能にする。

## 仕様
### 機能要件
- フィルター状態 `filters.dayOfWeek` に特別な値 `'TODAY'` を許容する。
- フィルタリング実行時、`'TODAY'` が選択されている場合は、現在の日付から算出される曜日コード（`MON`-`SUN`）に動的に変換して判定を行う。
- `localStorage` に `'TODAY'` が保存された場合、再読み込み時にも `'TODAY'` として解決され、その日の曜日が適用される。

### UI/メッセージ
- フィルターモーダルの「曜日」セクション内、「すべて選択」「解除」の横に「今日のみ」ボタンを追加する。
- **「今日のみ」（または「今日」が含まれる場合）は、ボタンの背景色やテキスト色を変更して選択状態であることを明示する。**
- フィルタータグには従来通り、選択されている場合は「今日」を表示する。

### 制約・前提
- クライアントサイドの `new Date()` を使用して「今日」を判定する。

## 実装計画
### 使用するクラス・ファイル
- UIロジック: `src/view.html`

### 処理フロー
1. 曜日コードの配列 `['SUN', 'MON', ...]` を定義する。
2. `filteredData` の `useMemo` 内で、`filters.dayOfWeek` に `'TODAY'` が含まれる場合、現在の曜日コードを追加した `effectiveDays` を作成する。
3. UI側の `filterOptions.dayOfWeek` に `'TODAY'` を明示的に追加する（または「今日」ボタンを別枠で作成する）。
4. フィルタータグの表示ロジック (`activeFilterTags`) で `'TODAY'` を「今日」と表示するように修正する。
5. 初期状態（`getInitialFilters`）のデフォルト値を `['TODAY']` に設定することを検討する。

### 技術的な判断・注意点
- `'TODAY'` と他の曜日（例：'MON'）を同時に選択することも可能とする。
- 保存されるのは `'TODAY'` という文字列であるため、日付が変わっても自動的に追従する。
